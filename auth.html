How Authentication Works
  • Public Endpoints: No authentication required (e.g., /api/public/customers)
  • Protected Endpoints: Require JWT token in Authorization header (e.g., /api/customers)
  • Login: POST credentials to /api/auth/login to receive a JWT token
  • Token Usage: Include token as Bearer {token} in Authorization header
  • Token Storage: Stored in browser localStorage
